# Lakehouse Layers

Phase 2 uses a local medallion architecture: bronze, silver, and gold.

## Bronze

Bronze is the raw append-style landing layer.

In this project, bronze contains CSV files generated by local Python scripts:

- `data_lake/bronze/historical_consumption/`
- `data_lake/bronze/weather/`
- `data_lake/bronze/customer_metadata/`
- `data_lake/bronze/meter_metadata/`
- `data_lake/bronze/tariff/`

Bronze keeps source-like imperfections:

- string timestamps
- string numeric fields
- missing values
- duplicate records
- invalid regions

This mirrors professional platforms where raw source data is preserved for traceability.

## Silver

Silver is the cleaned, typed, validated layer.

In this project, silver contains Parquet outputs:

- `data_lake/silver/clean_consumption/`
- `data_lake/silver/clean_weather/`
- `data_lake/silver/clean_customer/`
- `data_lake/silver/clean_meter/`
- `data_lake/silver/clean_tariff/`
- `data_lake/silver/rejected_records/`

Silver is where data becomes dependable enough for downstream transformation.

## Gold

Gold is the business-ready curated layer.

In this project, gold contains Parquet outputs:

- `data_lake/gold/daily_region_consumption/`
- `data_lake/gold/daily_customer_consumption/`
- `data_lake/gold/monthly_region_consumption/`
- `data_lake/gold/consumption_weather_features/`
- `data_lake/gold/customer_usage_summary/`

Gold tables are shaped for analytics, BI, and warehouse loading.

## Azure Mapping

| Local Layer | Azure-Style Equivalent |
| --- | --- |
| local `data_lake/bronze` folders | ADLS Gen2 bronze container or zone |
| local `data_lake/silver` Parquet | Databricks/Synapse silver Delta tables |
| local `data_lake/gold` Parquet | Curated gold Delta tables |
| SQL Server Developer Edition | Azure SQL Database or Synapse serving layer |

Phase 2 uses Parquet locally. Delta Lake can be introduced later when the project moves closer to a Databricks-style implementation.
